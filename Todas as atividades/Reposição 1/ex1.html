Diferenças entre CSS Inline, Interno e Externo

Inline

O CSS Inline é aplicado dentro de uma tag específica no documento HTML usando o atributo "style". Essa prática não é recomendada, já que é mais complicado atribuir estilos individualmente para tags que seguem um mesmo padrão, por exemplo. A manutenção também pode demorar mais com o uso desse tipo de estilização. Os casos em que pode ser útil são em páginas em que você não tem acesso ao arquivo de CSS ou quando precisar aplicar estilo em apenas um elemento. Exemplo:

HTML

<p style="color: red;">TEXTO AQUI</p>

Interno

O CSS Interno é adicionado na seção head da página HTML, aplicando todas as características de estilo dentro da tag <style></style>. Nesse caso não há necessidade de carregar vários arquivos (HTML e CSS esão no mesmo arquivo), entretanto, apenas a página que contém esses estilo é afetada e pode haver o aumento do tempo de carregamento da página. Exemplo:

HTML

<head>
    <style>
        p{
            color: blue;
        };
    </style>
</head>
<body>
    <p>TEXTO AQUI</p>
</body>

Externo

O CSS Externo é a maneira mais conveniente de estilizar as páginas HTML. O CSS, nesse caso, fica em um arquivo separado e afeta todas as páginas que o tiverem como referência. Deve-se fazer uma referência a um arquivo ".css" dentro da seção head através da tag <link>. Exemplo:

HTML

<head>
    <link rel="stylesheet"
    type="text/css"
    href="estilo.css">
</head>
<body>
    <p>TEXTO AQUI</p>
</body>

estilo.css

p{
    color: gold;
};

Tabela comparativa

Tipo Vantagens Desvantagens
Inline
  • Útil se você quiser testar e visualizar alterações.
  • Útil para correções rápidas.
  • Reduzir solicitações HTTP.
  • CSS Inline deve ser aplicado a cada elemento.
Interno
  • Apenas uma página é afetada pela pagina de estilos.
  • Classes e IDs podem ser usados ​​por stylesheet interno.
  • Não há necessidade de carregar vários arquivos. HTML e CSS podem estar no mesmo arquivo.
  • Aumento do tempo de carregamento da página.
  • Ela afeta apenas uma página – não é útil se você quiser usar o mesmo CSS em vários documentos.
Externo
  • Tamanho menor de páginas HTML e estrutura mais limpa.
  • Velocidade de carregamento mais rápida.
  • O mesmo arquivo ".css" pode ser usado em várias páginas.
  • Até que o CSS externo seja carregado, a página pode não ser processada corretamente
